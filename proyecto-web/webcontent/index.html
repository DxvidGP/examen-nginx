Componentes
🔀

Proxy Inverso

nginx escucha en el puerto 80 del host. Distribuye el tráfico en round-robin e inyecta la cabecera X-Backend-Server para identificar qué instancia respondió.

⚙️

Backends Apache

Dos contenedores httpd:alpine sirven contenido estático desde el volumen compartido. No exponen ningún puerto al exterior.

🔒

Red Aislada

Los backends viven en la red interna webnet. Solo el proxy tiene acceso a ambas redes, actuando de DMZ.

📦

Volumen Compartido

El contenido web reside en un volumen Docker montado en ambos backends. Actualiza una sola vez, sirve desde todas las instancias.

Diagrama

Arquitectura de red interna

El proxy nginx es el único contenedor accesible desde el host. Los backends residen en la red webnet, completamente aislados del exterior. El volumen webdata garantiza consistencia de contenido entre ambas instancias.
